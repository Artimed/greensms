# Active Constraints (MVP)

> Status: `ACTIVE`
> Scope: обязательные архитектурные и продуктовые ограничения
> Updated: `2026-03-15`
> Source: `zelenaya_sms_tz_v1.md` + `design_v2_single_page.md` + `pro_architecture.md` + текущая реализация

## 1. Product Scope (Locked)
- Приложение локальное, без облака и без логина.
- UI одностраничный: главный экран + полноэкранный QR-режим + экран настроек через иконку.
- Источник SMS: только последние 10 сообщений.
- Баланс не вычисляется, а берётся из последнего релевантного SMS.

## 2. Architecture Scope (Locked)
- Хранилище: `SQLite` (`sqflite`).
- Native bridge: Flutter + Kotlin (`MethodChannel` / `EventChannel`).
- QR: внутренний payload формата `greensms://qr/v3/` с HMAC-подписью и backward compatibility `v2`.

## 3. Permissions Policy (Locked)
- Разрешены: `READ_SMS`, `RECEIVE_SMS`, `CAMERA`, `SEND_SMS`.
- `SEND_SMS` работает только как opt-in режим через настройку `directSmsEnabled` (по умолчанию `OFF`).
- При включенном `directSmsEnabled` приложение сначала пытается отправить SMS на `900` напрямую.
- При ошибке прямой отправки выполняется fallback в системный `sms:` composer.
- Для Сбера команда формируется на номер `900` в формате `ПЕРЕВОД <phone> <amount> [last4]`.
- Маршрутизация банка: приоритет у ручного выбора пользователя (`страна` → `банк`) в настройках; если не выбрано — fallback на primary bank по locale устройства.
- Поддерживаются 2 режима генерации QR:
  - `My QR`: только телефон (без суммы);
  - `QR с суммой`: телефон + обязательная сумма.
- Поддерживается режим перевода **без QR**: пользователь вводит телефон + сумму (и опционально `last4` источника), после чего сразу открывается подтверждение и отправка платежной команды.
- В Sber-flow перед формированием SMS сумма всегда обязательна (если в QR её нет — вводится на стороне сканирующего), `last4` источника — опционален и задаётся раскрывающимся блоком.

## 4. Current MVP+ Additions (Already Implemented)
- Дневной лимит переводов внутри приложения (`daily_limit`).
- Учет лимита по QR-событиям со статусами `prepared/sent/cancelled`; в лимит включается только `sent`.
- Карточка статуса лимита на главном экране.
- Инфо-экран правил и лимитов переводов.
- Расширенный парсер SMS и набор тестов для банковских шаблонов.
- Расширенный settings-flow: язык, тема, лимит, разрешения, обслуживание локальных данных.

## 5. Distribution Model (Locked 2026-03-15)

Два отдельных APK из одного репозитория через Android Product Flavors:

| Flavor | Package | Парсер | Интернет |
|--------|---------|--------|----------|
| `free` | `com.greensms.free` | Bundled, не обновляется | ❌ |
| `pro`  | `com.greensms.pro`  | OTA с Supabase | ✅ |

- Free — стабильная базовая версия, выходит редко, только при кардинальных изменениях.
- Pro — активно поддерживается, платная (ежемесячная подписка).
- Код Pro-фич (`forwarding/`, `pro_license/`, `push/`) не компилируется в Free APK.
- Донаты — только в Free (кнопка в About). В Pro нет.

## 6. Explicit Non-Goals (MVP)
- Банковская интеграция / API банков.
- Полноценная бухгалтерия и аналитика.
- Автоматические платежи или процессинг транзакций.
